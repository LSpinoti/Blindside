# x402

> HTTP-native payments for APIs and AI agents.

`x402` is a protocol pattern that uses HTTP `402 Payment Required` so an API can
charge per request and an agent can pay automatically before retrying the same
call.

In short:

- `x402` pays for access to tools/data/services.
- `Unlink` moves user/business funds privately on-chain.

Use both together: pay agents/tools with `x402`, settle confidential payroll
and treasury flows with Unlink.

## Why it matters here

This project focuses on private finance apps and autonomous agents. `x402`
enables agentic pay-per-call infrastructure:

- No API key billing lock-in for every integration
- Machine-to-machine micropayments for premium endpoints
- Verifiable payment receipts for audit trails

## Standard request flow

1. Agent calls a paid API endpoint.
2. API responds with `402 Payment Required` and payment requirements.
3. Agent wallet prepares payment proof/authorization.
4. Agent retries the same request with payment data attached.
5. API verifies payment and returns the requested result.

## Ghost Payroll Ops mapping

Ghost Payroll Ops can use `x402` for paid intelligence, while Unlink performs
private value transfer:

- `Compliance Agent`: sanctions/KYC/tax rule lookups via paid APIs
- `FX Agent`: real-time conversion quotes from paid market data APIs
- `Risk Agent`: anomaly/fraud scoring from paid detection services
- `Payout Agent`: executes private payouts through Unlink

Separation of concerns:

- `x402`: buy external decisions/data
- `Unlink`: execute confidential transfers/deposits/withdrawals
